优化app功能
- 移除首页的export srt按钮和相关代码，因为history页面已经有这个功能了
- 修改首页的"弹出预览"按钮，改为toggle的交互和显示逻辑来打开和关闭浮窗
- 在设置页面添加快捷键区域，支持如下几个功能（默认值都未绑定键位）：开始/停止转录，开启/关闭翻译，打开/关闭浮窗，隐藏/显示整个窗口

追加优化
- 快捷键录制改用 NSEvent.addLocalMonitorForEvents 实现可靠按键捕获，支持 ESC 取消录入
- "弹出预览"按钮图标改为 `rectangle.dock`
- 设置页面语音模型区域固定最大高度 200pt，内部滚动列表
- 模型操作按钮文案从"下载"改为"选择"
- "就绪"状态改为绿色药丸形徽章（✓ Ready）

全局快捷键（CGEvent tap）
- 抽取 `GlobalHotkeyManager` 单例管理全局快捷键的生命周期
- 使用 `CGEvent.tapCreate` 创建系统级事件拦截（替代之前的 NSEvent 双监听方案，后者在 app 非焦点时不可靠）
- CGEvent tap 运行在 session 级别，能真正拦截和消费匹配的按键事件
- 线程安全的 `cachedBindings` 静态缓存，避免 C 回调中的主线程隔离问题
- 自动重试（最多 5 次）、健康检查（每 30s）、权限轮询机制确保稳定性
- 需要 macOS 辅助功能权限，设置页面动态显示权限状态和授权引导
